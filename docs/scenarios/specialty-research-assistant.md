# 专病科研助手

Owner: `opl-health-platform`
Purpose: `specialty_research_scenario_reference`
State: `candidate_first_pilot`
Machine boundary: 本文是第一试点建议，不代表专病、科室 owner、资料边界、实现、运行、交付或验收已经确定。

专病科研助手是 OPL Health Platform 的第一试点建议场景。

它面向医生、研究者和科室科研团队，帮助他们围绕一个专病方向完成文献证据、研究问题、变量定义、分析计划、图表报告和论文材料的持续推进。

## 用户

- 临床医生。
- 科室科研骨干。
- 研究生和科研助理。
- 科室负责人。

## 典型输入

- 专病方向或研究问题。
- 临床指南、专家共识和关键文献。
- 研究方案、病例定义、变量列表。
- 已有数据字典、统计结果或图表。
- 科室历史报告、论文草稿或项目材料。

## 需要的医学知识

- 专病指南和专家共识。
- 关键综述和高质量研究。
- 疾病定义、诊断标准、分型标准。
- 结局指标、暴露因素和协变量定义。
- 研究设计和统计方法参考资料。

## 需要的临床规则

- 纳入和排除标准。
- 变量口径和编码规则。
- 风险分层或亚组规则。
- 分析计划和结果解释边界。
- 引用、图表、统计和报告审查规则。

## 需要的工具

- 文献检索和引用管理。
- 表格、统计和图表工具。
- 报告、论文、幻灯片生成工具。
- 科室资料库或项目资料库。
- 可选的结构化科研数据库。

## 智能体工作流

```text
研究问题澄清
-> 文献与指南证据整理
-> 变量和纳排标准定义
-> 分析计划或报告大纲
-> 图表和结果解释
-> 报告或论文材料生成
-> 审查与修改
-> 交付记录
```

## 人工确认点

- 研究问题和适用人群。
- 纳入排除标准。
- 变量定义和结局指标。
- 统计方法选择。
- 结果解释和结论边界。
- 对外提交材料。

## 交付物

- 文献证据摘要。
- 研究方案草案。
- 变量定义表。
- 分析计划。
- 图表说明。
- 报告或论文段落。
- 审查记录和继续入口。

## 审查方式

OPL Health Review 重点检查：

- 引用是否可追溯。
- 指南和文献是否匹配研究问题。
- 变量口径是否一致。
- 统计方法和图表解释是否越界。
- 交付物是否保留来源和负责人。

## 与 OPL Cloud 的关系

- OPL Workspace 提供科研任务工作空间。
- OPL Gateway 提供模型能力和用量记录。
- OPL Fabric 接入文献、资料库、统计和报告工具。
- OPL Ledger 记录任务过程、输入来源、产物和审查结果。
- OPL Console 管理科室成员、权限、工作空间和试点资源。
