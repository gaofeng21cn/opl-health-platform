# 随访管理助手

随访管理助手面向医生、护士和患者管理团队，帮助他们制定随访计划、整理随访记录、识别需要人工关注的情况，并生成阶段总结。

这个场景商业价值高，但涉及患者触达、医疗责任、隐私和风险提示，适合作为后置规划。

## 用户

- 临床医生。
- 专科护士。
- 患者管理团队。
- 科室运营或管理团队。

## 典型输入

- 患者分组或随访名单。
- 诊疗记录摘要。
- 出院小结或病程资料。
- 随访规范和宣教材料。
- 既往随访记录。

## 需要的医学知识

- 专病随访指南。
- 风险分层资料。
- 患者教育材料。
- 常见问题和处理建议边界。

## 需要的临床规则

- 随访周期。
- 风险分层。
- 提醒规则。
- 异常情况升级规则。
- 人工确认和医生介入规则。

## 需要的工具

- EMR 或病例摘要引用。
- 随访系统。
- 消息或提醒工具。
- 表格和报告工具。
- 患者管理数据源。

## 智能体工作流

```text
随访对象和目标确认
-> 风险分层
-> 随访计划生成
-> 随访记录整理
-> 异常情况标记
-> 人工确认
-> 阶段总结
```

## 人工确认点

- 患者分组和触达范围。
- 风险分层。
- 异常情况。
- 对患者发送的内容。
- 需要临床处理的建议。

## 交付物

- 随访计划。
- 随访记录摘要。
- 异常情况清单。
- 阶段总结。
- 需要医生确认的事项。
- 审查记录。

## 审查方式

OPL Health Review 重点检查：

- 是否遵循医院数据和患者触达策略。
- 是否明确人工确认点。
- 是否保留异常情况和医生介入记录。
- 是否避免把智能体输出直接作为最终医疗决策。

## 与 OPL Cloud 的关系

- OPL Console 需要支持更严格的权限、审计和人工确认策略。
- OPL Fabric 需要支持随访系统、患者管理工具和消息通道的受控接入。
- OPL Ledger 需要记录触达边界、人工确认和异常升级记录。
- OPL Workspace 展示随访计划、异常清单和总结材料。
