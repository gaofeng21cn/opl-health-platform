# 合规与责任边界

OPL Health Platform 通过 OPL Health Review 把能力边界、审查边界和责任边界讲清楚。

## 基本原则

- 医疗智能体服务于专业人员工作，不替代医生作出最终医疗决策。
- 患者级数据默认遵循医院的数据安全、权限、审计和脱敏策略。
- 医学知识、临床规则和工具调用需要记录来源、版本、适用范围和负责人。
- 关键任务需要保留输入来源、运行过程、审查结果和交付记录。
- 高风险输出需要人工确认点。

## 审查对象

| 对象 | 审查重点 |
| --- | --- |
| 医学知识包 | 来源、版本、适用范围、更新机制 |
| 临床规则包 | 规则依据、使用场景、人工确认点 |
| 医疗工具包 | 权限、数据范围、调用记录 |
| 专病模板 | 输入要求、输出格式、审查路径 |
| 医疗智能体 | 能力边界、失败模式、版本变化 |
| 任务交付物 | 来源、计算、引用、审查结果 |

## 证据记录

重要任务应留下可复查记录：

```text
输入来源
→ 使用的知识和规则
→ 工具调用和运行环境
→ 输出产物
→ 审查结果
→ 负责人
→ 后续继续入口
```

这些记录用于质量管理、复盘、审计和后续接力。
