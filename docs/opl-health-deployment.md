# OPL Health Deployment

OPL Health Deployment 是 OPL Health Platform 的医院部署能力层。

它面向医院的信息化、安全、权限、数据和计算资源条件，规划院内私有化、专有云、混合部署和科室试点路径。

## 部署形态

| 模式 | 适用场景 | 特点 |
| --- | --- | --- |
| 院内私有化 | 数据敏感、内网要求强、医院具备基础设施 | 系统和数据主要留在院内 |
| 专有云 | 医院希望托管运行，同时要求专属资源和隔离策略 | 平衡运维效率和资源隔离 |
| 混合部署 | 模型、计算、存储或系统分布在不同环境 | 按任务和数据边界组合资源 |
| 科室试点 | 单科室、单专病、单科研团队先行 | 小范围验证价值和流程 |

## 接入对象

- 医院账号和组织结构。
- 权限、审计和安全系统。
- HIS、EMR、LIS、PACS。
- 文献库、指南库和科研数据库。
- 院内对象存储、文件系统或数据库。
- Docker、虚拟机、GPU、高性能计算或托管工作节点。

## 部署记录

每个部署方案应记录：

- 部署范围。
- 数据边界。
- 系统接入清单。
- 计算和存储资源。
- 模型和工具策略。
- 账号、权限和审计要求。
- 试点场景和验收标准。

## 与 OPL Cloud 的关系

OPL Cloud 提供工作空间、管理端、模型接入、资源底座和证据记录。OPL Health Deployment 把这些通用能力映射到医院的实际部署路径。
