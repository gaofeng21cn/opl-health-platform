# 医疗智能体生命周期

医疗智能体需要从设计、测试、审批、部署到运行和复查形成闭环。

```text
场景定义
→ 智能体蓝图
→ OPL Health 能力绑定
→ 测试与审查
→ 发布审批
→ 工作空间启用
→ 任务运行
→ 证据记录
→ 版本更新
```

## 关键阶段

| 阶段 | 主要产物 | 负责人 |
| --- | --- | --- |
| 场景定义 | 目标用户、任务边界、输入输出、风险点 | 医疗产品负责人、科室专家 |
| 智能体蓝图 | 角色、能力、工具、规则、交付格式 | AI 团队、OMA |
| OPL Health 能力绑定 | Knowledge、Protocol、Tools、专病模板 | 医疗内容负责人、信息化团队 |
| 测试与审查 | 测试案例、失败案例、审查记录 | 科室专家、AI 团队 |
| 发布审批 | 版本、权限、资源、适用范围 | 医院管理端 |
| 工作空间启用 | 可用入口、账号、项目空间 | OPL Workspace / Console |
| 任务运行 | 任务过程、工具调用、产物 | 用户、智能体 |
| 证据记录 | 回执、来源、审查、继续入口 | OPL Ledger |
| 版本更新 | 变更说明、回归检查、重新审批 | 平台团队 |

## 最小版本

第一版只需要跑通一个医疗智能体从蓝图到工作空间启用的路径：

1. 选择一个明确场景。
2. 绑定一个 OPL Health Knowledge、一个 OPL Health Protocol 和必要的 OPL Health Tools。
3. 用固定测试案例做审查。
4. 在工作空间中启用。
5. 为任务留下可检查的交付记录。
