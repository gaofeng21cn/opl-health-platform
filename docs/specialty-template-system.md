# 专病模板体系

专病模板把医学知识、临床规则、医疗工具、审查要求和交付格式组织成高频任务入口。

它是 OPL Health Platform 从平台能力走向科室应用的关键形态。

## 模板结构

每个专病模板应包括：

- 专病名称。
- 目标用户。
- 典型任务。
- 输入要求。
- 输出格式。
- 绑定的医学知识包。
- 绑定的临床规则包。
- 绑定的医疗工具包。
- 审查规则。
- 交付物和继续入口。

## 模板类型

| 类型 | 典型任务 |
| --- | --- |
| 专病科研模板 | 选题、文献、变量、统计、论文或报告 |
| 专病质控模板 | 指标核查、流程改进、异常解释 |
| 专病随访模板 | 随访计划、提醒、记录整理、总结 |
| 病例整理模板 | 病程、检查、治疗、时间线和摘要 |
| 指南匹配模板 | 指南条目、适用条件、证据说明 |

## 第一版建议

第一版选择一个边界清楚、资料可获得、专家容易审查的专病或科研场景，先跑通：

```text
专病模板
-> 医疗智能体
-> 工作空间任务
-> 审查记录
-> 交付物
```
